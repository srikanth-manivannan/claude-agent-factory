# Documentation Skills

> Documentation generation, API references, diagrams, and changelogs. Charter source: [/TAXONOMY.md](../../TAXONOMY.md). All skills conform to
> [/standards/](../../standards/) and validate against
> [skill.schema.yaml](../../shared/schemas/skill.schema.yaml).

| Skill | Does | Maturity |
|-------|------|----------|
| [changelog-generator](changelog-generator/) | Use this to generate a Keep a Changelog entry from a set of changes. | 🔹 baseline |
| [doc-linter](doc-linter/) | Use this to check documentation for completeness and freshness. | 🔹 baseline |
| [doc-writer](doc-writer/) | Use this to write or refresh a README and docs from code and intent, following the documentation standard. | 🥇 gold |

> **Maturity:** 🥇 gold = production reference · 🔹 baseline = complete but concise ·
> ⏳ draft = planned. See the [maturity model](../../README.md#maturity-model). Skill
> anatomy reference: [/skills/architecture/README.md](../architecture/README.md).
