# Testing Skills

> Unit, integration, and end-to-end testing, coverage, and test data. Charter source: [/TAXONOMY.md](../../TAXONOMY.md). All skills conform to
> [/standards/](../../standards/) and validate against
> [skill.schema.yaml](../../shared/schemas/skill.schema.yaml).

| Skill | Does | Maturity |
|-------|------|----------|
| [coverage-gap-finder](coverage-gap-finder/) | Use this to find and fill meaningful test-coverage gaps. | 🔹 baseline |
| [load-test-designer](load-test-designer/) | Use this to design and run load and performance tests. | 🔹 baseline |
| [test-refactorer](test-refactorer/) | Use this to improve flaky, slow, or brittle tests. | 🔹 baseline |
| [unit-test-generator](unit-test-generator/) | Use this to generate thorough unit tests for a unit of code, covering happy, edge, and failure cases. | 🥇 gold |

> **Maturity:** 🥇 gold = production reference · 🔹 baseline = complete but concise ·
> ⏳ draft = planned. See the [maturity model](../../README.md#maturity-model). Skill
