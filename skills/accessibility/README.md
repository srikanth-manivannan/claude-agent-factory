# Accessibility Skills

> Accessibility audits, WCAG conformance, and inclusive UX. Charter source: [/TAXONOMY.md](../../TAXONOMY.md). All skills conform to
> [/standards/](../../standards/) and validate against
> [skill.schema.yaml](../../shared/schemas/skill.schema.yaml).

| Skill | Does | Maturity |
|-------|------|----------|
| [a11y-auditor](a11y-auditor/) | Use this to audit a UI against WCAG and report prioritized, actionable accessibility fixes. | 🥇 gold |
| [aria-annotator](aria-annotator/) | Use this to add correct ARIA roles and labels to markup to close accessibility gaps. | 🔹 baseline |
| [color-contrast-fixer](color-contrast-fixer/) | Use this to detect and fix color-contrast failures to meet WCAG AA. | 🔹 baseline |
| [keyboard-nav-checker](keyboard-nav-checker/) | Use this to verify and fix keyboard operability and focus order. | 🔹 baseline |

> **Maturity:** 🥇 gold = production reference · 🔹 baseline = complete but concise ·
> ⏳ draft = planned. See the [maturity model](../../README.md#maturity-model). Skill
> anatomy reference: [/skills/architecture/README.md](../architecture/README.md).
